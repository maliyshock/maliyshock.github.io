Technical requirements

To submit a coin or token for listing, you should provide us with technical information on it. The info should include general description of the technology, full node, detailed documentation for API node, production-ready solutions, etc. Before applying for listing, please download the full version of requirements in PDF below and read it through.

Download in PDF

You have to provide next 3 most important things:

  • Link to smart contract, for example:
    • ERC –  https://etherscan.io/token/0x3fd8f39a962efda04956981c31ab89fab5fb8bc8
    • QRC –  https://explorer.qtum.org/address/57931faffdec114056a49adfcaa1caac159a1a25
    • WAVES –  https://wavesexplorer.com/tx/GFiWx4dcceJXGWfHc53GMXEidpciUCN5tnHUwE18NEfY
    • NEO –  https://neotracker.io/asset/ceab719b8baa2310f232ee0d277c061704541cfb
  • Test coins for the final test
  • Specific requirements. For example: some minimal amount of coin must stay on account.

In case of all technical information provided according to requirements above, your coin can be added in 2 days.

You have to provide next 6 most important things:

  • Full node
  • Where to download & how to install node (it can be binary file for Ubuntu or git-hub repository with sources).
  • Ensure that sources has detailed description how to build and all related dependencies.
    • Dash example
    • sources –  https://github.com/dashpay/dash
    • binary files –  https://github.com/dashpay/dash/releases
    • manual for Unix build –  https://github.com/dashpay/dash/blob/master/doc/build-unix.md
  • Blockchain explorer
  • Wallet (web or desktop)
  • Test coins or access to testnet, faucets link to refill coins. Ex – https://testnet.manu.backend.hamburg/faucet

In case of all technical information provided according to requirements above, your coin can be added in 2 weeks (generally). In other cases, the process can take up to 1 month.

If your coin is a fork of some existing coin that mentioned above, please provide detailed information about differences between your coin and base coin (API, launching, transactions processing etc.).

Otherwise, if your coin is a unique one, please provide the following info:

  • Assets
    • Detailed documentation for node's API, primarily following methods:
      • Create a new address (account)
      • Send transaction
      • Get transaction by ID
      • Find all transactions by parameters - receiver address (account), time period, starting/ending block etc.
    • Documentation about notifications (if you have some) in your blockchain.
    • Whether there is a system of notifications on a new block or incoming transaction (like in Bitcoin).
    • Libraries or SDK
    • Available production-ready solutions for different programming languages.
  • Links to GitHub and supporting community - slack, reddit, gitter etc.
  • Full node.
    • Where to download, how to install and run on Windows and Linux (Ubuntu), documented command line arguments.
  • Blockchain explorer
  • Wallet app or web wallet
  • Test coins
    • If testnet is available - test wallet, faucets or other ways to receive test coins. Provide some amount of real coins for testing.
  • How many symbols after the comma is the minimal unit of cryptocurrency
  • The commission range for your currency in the internal network (if it’s not the same %, please tell us the algorithm)
